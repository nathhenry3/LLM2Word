Troubleshooting & System Prompts

If the conversion fails on the first try, instruct your LLM with the following prompt before generating the output you wish to convert: 

Output only Markdown and LaTeX for equations. Include all text from the previous output.
Use $...$ for inline math and $$...$$ for display math.
Each display equation must be written in its own separate block.
Do not place multiple equations inside a single block.
Do not use visual formatting tricks such as spacing, alignment, or multiple equations on one line.
Do not include separators or decorative characters.
Do not explain the output.
Troubleshooting & System Prompts

If a table does not parse cleanly, ask your LLM to regenerate only the table in one of these simple formats before pasting it into the Table Optimizer: 

Output only the data table, with no explanation before or after it.
Use either CSV or a GitHub-flavored Markdown table.
For CSV, include one header row, then one row per record.
Escape any field that contains commas or quotes by wrapping it in double quotes.
Inside quoted CSV fields, write a double quote as two double quotes.
For Markdown, include a header row and a separator row such as | --- | --- |.
Do not include bullet points, numbering, code fences, captions, or extra prose.
